# Study Density Log (MVP)

## 概要
iPhone Safariでホーム画面追加して使える、学習の「実時間」と「集中換算時間」を記録するPWAです。Firebase Authentication + Cloud Firestoreでユーザー別同期します。

## ファイル構成
- `index.html`: 画面骨組み（6画面 + 下部ナビ）
- `style.css`: ダークテーマUI
- `app.js`: 認証、Firestore CRUD、集計、描画、JSONバックアップ
- `manifest.json`: PWAマニフェスト
- `service-worker.js`: 静的ファイルキャッシュ
- `firestore.rules`: ユーザー分離セキュリティルール

## ローカル実行（Windows）
1. このフォルダを開く
2. `app.js` の `firebaseConfig` を自分のFirebase値へ置換
3. ローカルサーバーを起動（例）
   - `py -m http.server 5500`
4. `http://localhost:5500` を開く

## デプロイ
静的ファイルのみなので GitHub Pages / Cloudflare Pages / Netlify にそのまま配置可能。

### GitHub Pages更新反映（Service Worker）
- GitHub Pages更新後は、通常は次回起動または再読み込みで新しい版が読み込まれます。
- 本実装では Service Worker のキャッシュ名をバージョン管理し、`activate` 時に古いキャッシュを自動削除します。
- さらに更新検知時は、無限リロード防止付きで自動1回リロードします。
- `app.js` / `style.css` にはバージョンクエリを付与し、古いキャッシュ混在時も新しいファイルへ切り替わりやすくしています。
- JS/CSSはService Workerで長期保持せず、基本はネットワーク取得（失敗時のみフォールバック）にして、古いコード残留を避けています。
- それでも反映されない場合のみ、ブラウザのサイトデータ削除を試してください。

## Firebase設定
### Authentication
- Googleプロバイダを有効化。
- 承認済みドメインにデプロイドメインを追加。
- 現在のMVPではPC/通常ブラウザでの安定性を優先して `signInWithPopup` を使用。

### Firestore
- データは `users/{uid}/...` 配下のみ使用。
- `firestore.rules` を適用してください。

## Firestore Security Rules
`firestore.rules` を以下コマンドでデプロイ（Firebase CLI）:
```bash
firebase deploy --only firestore:rules
```

## バックアップ
- 設定画面でJSONエクスポート/インポート。
- 現在のMVPはFirestore保存 + service-workerによる静的ファイルキャッシュ + JSONバックアップを中心に実装。
- IndexedDBは**将来的なオフライン補助機能**（ローカルキュー/一時キャッシュ）として導入予定。
- 画面表示時は、Firestore由来のユーザー入力文字列をHTMLエスケープして描画し、意図しないHTML実行を避けます。

## MVPでできること
- 初期教科/ラベル投入
- 記録追加/編集/削除
- 今日/今週/今月/累計の実時間・集中換算
- 週目標と達成率
- 次テストまで日数
- 教科別/ラベル別/質別の時間配分
- PWA基本対応
- 管理機能は設定画面内のサブ画面に移動
- ラベル選択はチップ式複数選択
- ログアウトは設定画面から実行
- 週目標は `weekStartDate`（月曜起点）ごとに1件のみ保存（既存があれば更新）

## 変更後の確認ポイント
- 下部ナビが「ホーム / タイムライン / 追加 / 目標 / 設定」の5項目になっている
- 設定画面から「教科・教材・ラベル管理」を開け、戻るボタンで設定に戻れる
- 記録追加のラベル選択がチップUIで複数選択できる
- 目標画面で、今週目標・達成率に加えて今日/今週/今月/累計の実時間と集中換算時間が表示される
- 記録追加の「学習時間（分）」と終了時刻を入れると、開始時刻が自動計算される
- 記録追加画面にストップウォッチを追加（学習時間へ反映可能）
- 設定内の管理画面で教科・教材の並べ替え（↑↓）ができる
- 時間表示は「xx時間xx分」形式で統一される
- ホーム画面で教材ごとの累計（教材名/教科名/実時間/集中換算/回数/ページ/問題）を確認できる
- 目標画面のカレンダーで学習日・月間合計・自習室チェックを確認できる
- カレンダーの日付をタップすると、その日の記録一覧（教科/教材/分数/質/メモ）と「自習室に行った」チェックを表示・保存できる
- 一覧は「上が最新、下が過去」の順（新しい順）で表示
- 学習時間と目標時間は「時間 + 分」で入力可能

## スケジュール機能（通年ベース＋期間上書き）
- 「スケジュール」画面でトラッキング開始日と、通年のデフォルトタスク（例: 単語100個など）を設定できる。
- 「夏休み」「冬休み」のように名前付きの期間を複数追加でき、その期間中はデフォルトタスクの代わりに期間専用タスクが使われる。
- 毎日、その日時点で有効なタスクセットをチェックリストとして表示。チェックで完了にできる。
- 前日に未完了のタスクは、当日画面の先頭に赤字の「繰り越し」として表示される。
- トラッキング開始日から連続で全タスク完了した日数を「ストリーク」としてホーム画面とスケジュール画面に大きく表示。ストリークが途切れた場合は、いつ・何日で途切れたかも表示する。
- データは `users/{uid}/settings/schedule`（通年デフォルト設定）・`users/{uid}/schedulePeriods`（期間ごとの上書き）・`users/{uid}/scheduleDays/{date}`（日ごとのタスクスナップショットと完了状態）に保存。

## 教科と教材の関係
- 教材は必ずいずれかの教科に属する（設定 > 教科・教材・ラベル管理で教科を選んで登録）。
- 記録追加フォームでは、選択中の教科に属する教材だけが選択肢に表示される（教科を切り替えると教材リストも切り替わる）。
- 既存教材は管理画面の「編集」ボタンから名前・所属教科を変更できる。
- 旧データなどで教科が未設定の教材は、どの教科でも選択肢に表示される。

## UIテーマ
- ダーク基調の「上品な勉強部屋」テーマ。深いチャコール背景 + ティール/藍のアクセント、カード型レイアウト、下部ナビはアイコン付き。
- 見た目は `style.css` にほぼ集約されており、色はファイル冒頭の CSS変数（`--bg` / `--accent` など）で一括変更できる。

## 通知（現状の実装と制約）
- 設定画面の「通知設定」からブラウザ通知の許可と、通知したい時刻（時）を設定できる。
- アプリを開いた（またはタブがフォアグラウンドに戻った）タイミングで、その日まだ通知していなければ「今日のタスク」をローカル通知として表示する。
- **制約**: 静的ホスティング（バックエンドなし）のため、アプリを完全に閉じている状態での確実な毎朝の自動通知はできない。本当に「アプリを閉じていても毎朝届く」通知を実現するには、Firebase Cloud Messaging + Cloud Functions（Blazeプラン）によるサーバー側スケジュール送信が必要（今後の拡張案を参照）。
